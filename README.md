```markdown
# ref-citation-embedder

> 自动将参考文献嵌入 Word 论文正文（DOCX），生成可点击跳转的 REF 域代码。
> 适用于本科/硕士论文、期刊论文、文献综述、学位论文的引用格式化。

---

## 与 Zotero 有什么不同？

| | Zotero | ref-citation-embedder |
|---|---|---|
| 引用定位 | 用户手动在每段插入引用 | AI 自动分析段落，决定引用位置 |
| 输入 | 用户逐篇选择文献 | 用户只提供论文 + 文献列表 |
| 输出 | 格式化引注 | 完整 REF 域 + 跳转链接 + 重排文献列表 |
| 人工介入 | 每篇都要操作 | 一次性跑完 |

**Zotero 是"手动挡"，ref-citation-embedder 是"AI 自动挡"。**

---

> **适用场景**：中文学术论文（本科/硕士论文、期刊论文、文献综述、学位论文）。参考文献格式采用 `[N]` 编号体系，标题支持「参考文献」「主要参考文献」「【参考文献】」等中文学术语境写法。

## 双轨模式（核心功能）

**外部 AI 分析文献内容 → 产出引用计划 → 脚本执行格式化**

```
用户提供：论文.docx + 参考文献.txt
        ↓
外部 AI逐段分析正文
        ↓
AI 产出映射表：P19→#1, P24→#2, P27→#3
        ↓
脚本执行：插入 REF 域 + 重排文献列表 + 加书签
        ↓
输出：论文_已引用.docx
```

**AI 做阅读理解，脚本做格式化工序，各司其职。**

详细说明见 [`docs/DUAL_TRACK.md`](docs/DUAL_TRACK.md)

---

## 功能

- **双轨模式（核心）**：外部 AI 分析段落归属 → 脚本执行格式化，AI 做判断，脚本做格式
- **REF 域插入**：生成 Word 原生 REF 域代码（5-run 结构），支持 Ctrl+Click 跳转
- **项目编号体系**：按正文首次出现顺序重新编号，增删文献后 F9 可自动顺延
- **段中定位**：Run 拆分机制实现标点前精确插入（`text[1]，next`）
- **安全保护**：强制备份 + 只读加载 + 另存输出，绝不修改原文件

## 安装

```bash
pip install -r requirements.txt
```

## 快速开始

### 最常用：AI 分析后生成引用（双轨）

```bash
# 1. 让 AI 分析正文，生成引用计划
# 2. 将映射表传入脚本
python scripts/refcite.py 论文.docx --smap "5:1:。,12:2:。" --verify
```

### 已有手打 [N] 标记，直接替换

```bash
python scripts/refcite.py 论文.docx --replace --verify
```

### 自动关键词匹配（兜底，精度有限）

```bash
python scripts/refcite.py 论文.docx --auto --verify
```

## 三种匹配模式

| 模式 | 说明 | 精度 |
|------|------|------|
| 模式①：显式映射 | 用户/AI 指定每段对应哪篇文献 | **最高** |
| 模式②：主题词提示 | 用户提供段落主题词，脚本在限定范围内匹配 | 中等 |
| 模式③：自动关键词 | 脚本自行匹配，高频词可能集中 | 兜底 |
