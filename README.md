# ref-citation-embedder

> 自动将参考文献嵌入 Word 论文正文（DOCX），生成可点击跳转的规范引用标记（REF 域代码）。
> 适用于本科/硕士论文、期刊论文、文献综述、学位论文的引用格式化。

---

> **适用场景**：中文学术论文（本科/硕士论文、期刊论文、文献综述、学位论文）。参考文献格式采用 `[N]` 编号体系，标题支持「参考文献」「主要参考文献」「【参考文献】」等中文学术语境写法。英文论文及非 `[N]` 编号体系的文献支持有限，建议先用小样本测试。

## 功能

- **REF 域插入**：生成 Word 原生 REF 域代码（5-run 结构），支持 Ctrl+Click 跳转
- **三种匹配模式**：用户提供显式映射（最可靠）、主题词提示、纯自动关键词匹配（兜底）
- **双轨扩展**：外部 AI/搜索工具产出映射表，skill 负责格式化执行
- **项目编号体系**：按正文首次出现顺序重新编号，增删文献后 F9 可自动顺延
- **段中定位**：Run 拆分机制实现标点前精确插入
- **安全保护**：强制备份 + 只读加载 + 另存输出，绝不修改原文件

## 安装

```bash
pip install python-docx
```

Python 3.8+ 支持，无其他外部依赖。

## 快速开始

```bash
# 模式D：正文已有 [N] 手打标记，原位替换为 REF 域
python scripts/refcite.py 论文.docx --replace --verify

# 模式①：外部 AI 给出句级映射
python scripts/refcite.py 论文.docx --smap "5:1:。,12:2:。" --verify

# 模式③：自动关键词匹配（兜底）
python scripts/refcite.py 论文.docx --auto --verify
```

详细用法见 `SKILL.md`。

## 三种使用方式

### 模式①：显式引用计划（推荐）

用户明确指定每段对应哪篇文献，跳过自动匹配，精度最高。

### 模式②：主题词提示

用户提供段落主题词，skill 在限定范围内做关键词匹配。

### 模式③：纯自动匹配

用户不提供任何信息，skill 自行从文献标题提取关键词匹配。

### 双轨（扩展）

外部 AI 或搜索工具分析文献内容 → 产出映射表 → 传入模式①。

## 安全边界

| ✅ 可以修改 | ❌ 绝不修改 |
|-----------|-----------|
| 参考文献段落（重排） | 脚注/尾注 |
| 正文匹配段落（追加REF域） | 页眉/页脚/分节符 |
| numbering.xml（动态分配ID） | 样式表 |
| 书签 | 原文文本内容 |
