# 双轨用法 + 映射表接口

> 双轨 = 外部工具分析内容 + skill 执行格式化。搜索只是获取映射的**一种手段**，不是双轨的必要条件。

---

## 什么是双轨

双轨是模式①的扩展工作流：

```
外部工具（AI/搜索引擎/人工）→ 产出映射表 → skill 执行格式化
```

skill 只提供映射表接口，不做文献调研和内容理解。

## 使用建议

| 方面 | 说明 |
|------|------|
| **映射表质量** | 映射越准确，最终引用越可靠。建议通读确认 |
| **搜索结果** | 可作为参考，建议对关键文献交叉验证 |
| **文献编号** | 映射表编号需与文末参考文献列表一一对应 |
| **句级精度** | 建议精读目标段落后再决定插入位置 |
| **首次使用** | 先测试 2-3 段，确认效果后再处理全文 |
| **保存后校验** | 加 `--verify` 参数自动检查书签和 REF 域结构：`python refcite.py 论文.docx --smap "19:1:。" --verify` |

## 索引规则

段落索引 = `doc.paragraphs` 数组中的 0-based 位置，**包含所有段落**（空行、标题、封面等）。

```python
doc.paragraphs[0]  → 文档第 1 段
doc.paragraphs[19] → 文档第 20 段
```

过滤无效段落是 skill 内部的事，映射表只需给出原始索引。

## 映射表格式

### 段落级映射

```python
{19: 1, 24: 2, 27: 3}  # {段落索引: 文献编号}
```

### 句级映射（推荐）

```bash
python scripts/refcite.py 论文.docx --smap "14:2:；,14:1:。"
#                             段落:文献:标记,段落:文献:标记
```

标记字符说明：

| 标记 | 含义 | 示例结果 |
|------|------|---------|
| `。` | 句号前插入 | `text[1]。` |
| `，` | 逗号前插入 | `text[1]，` |
| `；` | 分号前插入 | `text[2]；` |

> ⚠️ **限制：每段仅匹配第一个标点**。如果某段有多个句号，`--smap` 只命中第一个。无法指定"第二个句号前插入"。需要精确控制句级位置的段落，请使用**场景E（双轨变体）**：AI 直接写入 `[N]` 手打标记 → `--replace` 消费，不受此限制。

支持任意字符，外部工具产什么标记，脚本就在哪里插。

## skill 侧行为

收到映射表后：
1. 验证段落索引是否在文档范围内
2. 过滤被跳过的段落（摘要/封面/标题等），并报告被跳过的条目
3. 按映射表直接插入 REF 域（跳过自动匹配）
4. 按首次出现顺序分配项目编号
5. 重排参考文献 + 加书签 + 保存
6. 报告未使用的文献

## 项目编号示例

```
映射表指定：P19→#1, P24→#2, P27→#3
处理后编号：
  正文首次出现 → [1]（P19 的 #1）
  第二次出现 → [2]（P24 的 #2）
  第三次出现 → [3]（P27 的 #3）
  参考文献列表按 [1][2][3] 重新排序
```

## 变体：已有部分引用，补充引用

当正文中已存在手打 `[N]` 标记（如 [1]~[6]），AI 分析后需补充更多引用时：

1. AI 逐段分析正文内容，判断各段落应引用哪篇文献
2. AI 在对应段落的句末位置写入新的 `[N]` 手打标记（如 `[7]`、`[15]`）
3. 运行 `--replace --keep-numbers`：
   - 原有标记 `[1]~[6]` 原位替换为 REF 域，编号不变
   - 新标记 `[7]~[16]` 同样原位替换，编号保持与文末文献列表一致
4. 未使用文献条目（如附录文本被误识别）仅出现在"未引用"报告中，不影响正文

### 使用要点

| 要点 | 说明 |
|------|------|
| **入口** | SKILL.md 场景E |
| **AI 角色** | 分析正文 → 决定引哪篇 → 写入 `[N]` 标记 |
| **脚本角色** | 找到 `[N]` → 替换为 REF 域 → 加书签 → 重排 |
| **编号策略** | `--keep-numbers`，已有编号不动，新编号保持原序号 |
| **附录处理** | 附录如被误识别为文献条目，不写 `[N]` 标记即可，不产生 REF 域 |
| **补救** | 用户可在 Word 中直接增删 `[N]` 标记后重跑 |

### 完整流程示例

```
输入：论文.docx（已有 [1]~[6]，参考文献列表 [1]~[16] + 附录）
  ↓
AI 分析：P56 企业背景 → 引 [15]，P77 RFM分层 → 引 [7][11]，……
  ↓
AI 写入标记：在对应段落句末插入 [7][11]、[8]、[9]…… 手打文本
  ↓
python refcite.py 论文.docx --replace --keep-numbers --verify
  ↓
输出：论文_已引用.docx（17 处 REF 域，16 个书签锚点，校验 PASS）
```

> 这是本次实际测试验证通过的流程。与标准双轨的区别在于：映射表不是通过 `--smap` flag 传入，而是由 AI 直接写入正文 `[N]` 标记，脚本通过 `--replace` 消费。**两种方式本质相同**（AI 分析 + 脚本格式化），只是传递映射表的载体不同。

## 外部 AI 对接指南

### 文档解析

```python
import docx
doc = docx.Document('论文.docx')
for i, p in enumerate(doc.paragraphs):
    print(f'[{i}] {p.text}')
```

### 段落分析

逐段判断：
1. 段落是否在讨论某篇文献的研究对象或观点？
2. 段落是否在比较多篇文献？
3. 段落是否引用某篇文献的方法或结论？

### 信息获取方式

AI 逐条判断对每篇文献的熟悉程度。对不熟悉的文献（新期刊、非公开方法、不确定结论），主动询问：

> "这篇《xxx》我不太确定核心内容，需要搜一下确认引用位置吗？"

用户同意则搜索，不同意则 AI 凭训练知识判断。如果所有文献都熟悉，直接分析即可，不需要询问。

搜索不是双轨的必要条件。AI 自行决定每条文献是否需要搜索，不让用户做全局决策。

### 句级分析（默认）

按句子分析，**逐句判断引用归属**。即使整段只引一篇文献，也定位到最相关的那句话的句末，不是段末。

跨文献对比段落（一段话讨论多篇文献），按句号/分号拆到句子级别：

```
段落 P14 两句：
  句1: "李长莉在《晚清上海社会的变迁》中将…转型；" → 文献 #2
  句2: "王笛以近代成都茶馆为个体研究对象…互动。" → 文献 #1
```

```bash
python refcite.py 论文.docx --smap "14:2:；,14:1:。"
```

### 常见注意事项

| 场景 | 建议 |
|------|------|
| 摘要段落 | 通常不含引用，排除 |
| 封面/标题行 | 排除 |
| 参考文献列表本身 | 排除 |
| 对文献不熟悉 | 搜索或用户提供信息补充 |
| 多篇文献在同一段 | 拆为句级映射 |
| 不确定映射是否准确 | 仅映射确定的部分，留空不映射 |
| **作者名 ≠ 需要引用** | 段落中出现作者名不代表内容出自该文献 |
| **传记信息不引** | 出生日期、职位等公开信息不属可引用内容 |
